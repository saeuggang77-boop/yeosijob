# 스탭 계정 구현 계획 (v3)

작성일: 2026-04-08
업데이트: 2026-04-08 v3 (경로 검증 + 보안 보완 + 크론 충돌 해결)
목적: 1인 운영자가 사장님 대신 광고를 무제한으로 등록할 수 있는 전용 계정. **공개 페이지에서는 일반 사장님 광고와 100% 동일하게 보이도록 한다.**

---

## 핵심 원칙

> **"겉으로 보이는 건 사장이 직접 올린 것처럼 보여야 한다."**
>
> 스탭 광고는 일반 유료 광고와 동일한 데이터 구조(90일 endDate, isVerified=true)로 저장되어, 공개 페이지(`/jobs`, `/jobs/[id]`, 메인 등)에서 일반 사장님 광고와 시각적으로 구분 불가능해야 한다.
>
> 결제만 안 하고, 개수 제한 없이, 유료/무료 모두 등록 가능하면 된다.

## 합의된 사양 (v3)

### 1. 계정 정보
- **이메일**: `staff@yeosijob.com`
- **역할**: `BUSINESS` (UserRole 변경 없음)
- **구분 플래그**: `isStaff: true`
- **사업자 인증**: `isVerifiedBiz: true`로 미리 세팅 (우회 목적)
- **사업자번호**: `null` (우회)
- **표시 이름**: "여시잡 스탭" (운영자 본인만 보는 정보)

### 2. 스탭 권한
- ✅ 무료/유료 광고 무제한 등록 (2건/5건 제한 우회)
- ✅ 유료 광고도 결제 없이 즉시 `ACTIVE` 활성화
- ✅ Rate limit 우회 (대량 등록 허용)
- ✅ 사업자 인증 체크 우회
- ✅ `totalAmount = 0` 으로 저장 → 매출 통계 영향 없음
- ✅ **`endDate = now + 90일`로 저장** (일반 유료 광고와 동일)
- ✅ **`totalPaidAdDays`는 누적하지 않음** (신규 사장님처럼 등급 없음 = 자연스러움)
- ✅ 자동 갱신 크론으로 D-1에 +90일 (최대 3회 = 약 360일)
- ✅ `maxEdits = 9999` (수정 무제한)
- ✅ 지원/리뷰는 일반 광고와 동일하게 받되 운영자가 무시 (별도 차단 안 함)
- ⚠️ **Payment 레코드는 생성하지 않음** (v3 결정) — 결제 목록 오염 방지
- ⚠️ **BANNER 상품은 운영자가 수동 관리** — 홈 12슬롯 점유 주의. 스탭이 대량 등록하면 진짜 사장님 광고가 밀림 → 운영 정책으로 절제
- ⚠️ **지원/리뷰 알림 폭탄 주의** — 스탭 본인이 알림 설정 페이지에서 수동으로 꺼야 함

### 3. 기존 사이트 영향
- **기존 BUSINESS 사용자**: `isStaff=false`(기본값) → 기존 로직 그대로 탐. 한 줄도 안 건드림
- **기존 광고**: 영향 없음
- **만료 크론 (`expire-ads`)**: **`user: { isStaff: false }` 필터 한 줄 추가** → 스탭 광고는 만료 크론이 건드리지 않음 (v3 변경)
- **알림 크론(notify-expiry)**: `user: { isStaff: false }` 필터 한 줄 추가
- **매출 통계**: `totalAmount=0`이라 자동 제외
- **공개 페이지**: 일반 사장님 광고와 동일하게 보임 (D-day, GradeBadge 없음, validThrough 등 모두 일반 광고와 동일)

### Step 3: `api/ads/route.ts` POST에 스탭 분기 추가

**파일**: `src/app/api/ads/route.ts`

#### 수정 위치 명확화

기존 코드는 `checkRateLimit`(line 90)이 `user 조회`(line 96)보다 먼저 실행됨. 스탭 분기는 rate limit 이전에 진입해야 하므로, **user 조회를 line 90 위로 이동**해야 함.

#### 새로운 코드 순서

```ts
export async function POST(request: NextRequest) {
  try {
    const session = await auth();
    if (!session || session.user.role !== "BUSINESS") {
      return NextResponse.json({ error: "권한이 없습니다" }, { status: 401 });
    }

    // ⬇ user 조회를 위로 이동 + isStaff 추가
    const user = await prisma.user.findUnique({
      where: { id: session.user.id },
      select: { isVerifiedBiz: true, businessNumber: true, isStaff: true },
    });

    // ⬇ Feature flag
    const STAFF_ADS_ENABLED = process.env.ENABLE_STAFF_ADS !== "false";

    // ⬇ 스탭 분기 (rate limit 및 모든 제한 이전에 진입)
    if (user?.isStaff && STAFF_ADS_ENABLED) {
      return await handleStaffAdCreation(request, session.user.id);
    }

    // ⬇ 아래는 기존 로직 (한 줄도 안 건드림)
    const { success } = await checkRateLimit(`ad:${session.user.id}`, 3, 60_000);
    if (!success) { ... }

    if (!user?.isVerifiedBiz) { ... }

    // ... (이하 기존 로직 그대로)
  }
}
```

#### 스탭 광고 생성 함수 (v3: Payment 블록 삭제)

```ts
async function handleStaffAdCreation(request: NextRequest, userId: string) {
  const body = await request.json();
  const {
    businessName, businessType, contactPhone, contactKakao, contactTelegram,
    address, addressDetail, locationHint,
    title, salaryText, workHours, benefits, description,
    workEnvironment, safetyInfo,
    regions, districts = [],
    productId, options = [], optionValues = {},
    bannerColor, bannerTitle, bannerSubtitle, bannerTemplate,
    detailImages = [],
  } = body;

  // 필수 필드 검증
  if (!businessName || !businessType || !contactPhone || !title || !salaryText || !description) {
    return NextResponse.json({ error: "필수 항목을 모두 입력해주세요" }, { status: 400 });
  }

  const product = AD_PRODUCTS[productId];
  if (!product) {
    return NextResponse.json({ error: "올바른 상품을 선택해주세요" }, { status: 400 });
  }

  // BANNER가 아닌 경우 지역 필수
  if (productId !== "BANNER" && (!regions || regions.length === 0)) {
    return NextResponse.json({ error: "노출 지역을 선택해주세요" }, { status: 400 });
  }

  const now = new Date();
  const endDate = new Date(now);
  endDate.setDate(endDate.getDate() + 90); // 일반 유료 광고와 동일하게 90일

  const ad = await prisma.$transaction(async (tx) => {
    const newAd = await tx.ad.create({
      data: {
        userId,
        businessName,
        businessType: businessType as BusinessType,
        contactPhone: contactPhone.replace(/-/g, ""),
        contactKakao: contactKakao || null,
        contactTelegram: contactTelegram || null,
        address: address || null,
        addressDetail: addressDetail || null,
        locationHint: locationHint || null,
        title,
        salaryText,
        workHours: workHours || null,
        benefits: benefits || null,
        description,
        workEnvironment: workEnvironment || null,
        safetyInfo: safetyInfo || null,
        regions: regions as Region[],
        districts: Array.isArray(districts) ? districts : [],
        productId: productId as AdProductId,
        durationDays: 90,
        totalAmount: 0,        // 매출 통계 제외
        status: "ACTIVE",      // 즉시 활성화
        startDate: now,
        endDate,               // 90일 후 (자동 갱신 크론이 D-1에 +90일)
        autoJumpPerDay: product.autoJumpPerDay,
        manualJumpPerDay: product.manualJumpPerDay,
        maxEdits: 9999,        // 수정 무제한
        isVerified: true,
        autoRenewCount: 0,     // 자동 갱신 횟수 (최대 3회)
        bannerColor: bannerColor ?? 0,
        bannerTitle: bannerTitle || null,
        bannerSubtitle: bannerSubtitle || null,
        bannerTemplate: bannerTemplate ?? 0,
        detailImages: Array.isArray(detailImages) ? detailImages.slice(0, 10) : [],
      },
    });

    // 옵션 생성
    if (options && options.length > 0) {
      await tx.adOption.createMany({
        data: (options as string[]).map((optId: string) => ({
          adId: newAd.id,
          optionId: optId as AdOptionId,
          value: optionValues[optId] || null,
          durationDays: 90,
          startDate: now,
          endDate,
        })),
      });
    }

    // ⚠️ v3: Payment 레코드 생성 블록 삭제 — 관리자 결제 목록 오염 방지

    return newAd;
  });

  return NextResponse.json({ adId: ad.id, staff: true }, { status: 201 });
}
```

**중요**:
- `totalPaidAdDays`는 누적하지 않음
- Payment 레코드는 생성하지 않음 (v3 결정)
- audit이 필요하면 Ad의 `userId` + `isStaff: true`로 역조회 가능

### Step 4: 만료 크론에서 스탭 광고 제외 (v3 신규)

**파일**: `src/app/api/cron/expire-ads/route.ts`

기존 쿼리에 `user: { isStaff: false }` 한 줄 추가:

```ts
// 만료 대상 광고 조회

  where: {
    status: "ACTIVE",
    endDate: { lt: now },
    user: { isStaff: false },  // ⬅ v3 추가: 스탭 광고는 건드리지 않음
  },
  // ...

```

**이유**: 매시간 실행되는 `expire-ads`가 스탭 광고를 건드리지 않도록 함. 스탭 광고의 endDate 관리는 **`auto-renew-staff` 크론이 단독으로** 담당 → 크론 타이밍 충돌 원천 차단.

**주의**: 만약 autoRenewCount가 3에 도달한 스탭 광고는 `expire-ads`가 영원히 안 건드리므로 DB에 ACTIVE 상태로 남음. 이는 의도된 동작 — 운영자가 수동으로 삭제/취소.

### Step 5: 자동 갱신 크론 신규 추가

**신규 파일**: `src/app/api/cron/auto-renew-staff/route.ts`

```ts
import { NextRequest, NextResponse } from "next/server";
import { prisma } from "@/lib/prisma";

const MAX_AUTO_RENEW = 3; // 최대 3회 = 약 360일 후 정말 만료
const RENEW_DAYS = 90;

export async function GET(request: NextRequest) {
  // CRON_SECRET 검증
  const authHeader = request.headers.get("authorization");
  if (authHeader !== `Bearer ${process.env.CRON_SECRET}`) {
    return NextResponse.json({ error: "Unauthorized" }, { status: 401 });
  }

  const now = new Date();
  const tomorrow = new Date(now);
  tomorrow.setDate(tomorrow.getDate() + 1);
  const dayAfterTomorrow = new Date(now);
  dayAfterTomorrow.setDate(dayAfterTomorrow.getDate() + 2);

  // 스탭 광고 중 D-1인 것 찾기 (endDate가 내일 ~ 모레 사이)
  // Step 4 덕분에 expire-ads가 스탭 광고를 안 건드리므로 status: ACTIVE만 체크하면 됨
  const adsToRenew = await prisma.ad.findMany({
    where: {
      status: "ACTIVE",
      autoRenewCount: { lt: MAX_AUTO_RENEW },
      endDate: {
        gte: tomorrow,
        lt: dayAfterTomorrow,
      },
      user: {
        isStaff: true,
      },
    },
    select: { id: true, endDate: true, autoRenewCount: true },
  });

  let renewedCount = 0;
  for (const ad of adsToRenew) {
    const newEndDate = new Date(ad.endDate!);
    newEndDate.setDate(newEndDate.getDate() + RENEW_DAYS);

    await prisma.$transaction([
      prisma.ad.update({
        where: { id: ad.id },
        data: {
          endDate: newEndDate,
          autoRenewCount: { increment: 1 },
          lastAutoRenewedAt: now,
        },
      }),
      // AdOption도 같이 연장
      prisma.adOption.updateMany({
        where: { adId: ad.id },
        data: { endDate: newEndDate },
      }),
    ]);
    renewedCount++;
  }

  return NextResponse.json({
    success: true,
    renewedCount,
    timestamp: now.toISOString(),
  });
}
```

#### 7-2. 스탭 대시보드 렌더 함수

```ts
async function renderStaffDashboard(
  userId: string,
  user: { businessNumber: string | null; isVerifiedBiz: boolean; bizOwnerName: string | null },
  opts: { statusFilter: string; searchQuery: string; currentPage: number; pageSize: number }
) {
  const { statusFilter, searchQuery, currentPage, pageSize } = opts;

  const where: Record<string, unknown> = { userId };
  if (statusFilter !== "ALL") {
    where.status = statusFilter;
  }
  if (searchQuery) {
    where.OR = [
      { title: { contains: searchQuery, mode: "insensitive" } },
      { businessName: { contains: searchQuery, mode: "insensitive" } },
    ];
  }

  const [ads, total, activeCount, totalViewsAgg] = await Promise.all([
    prisma.ad.findMany({
      where,
      orderBy: { createdAt: "desc" },
      skip: (currentPage - 1) * pageSize,
      take: pageSize,
      select: {
        id: true, title: true, status: true, productId: true,
        totalAmount: true, viewCount: true, startDate: true, endDate: true,
        createdAt: true, manualJumpPerDay: true, manualJumpUsedToday: true,
        lastManualJumpAt: true, autoRenewCount: true,
      },
    }),
    prisma.ad.count({ where }),
    prisma.ad.count({ where: { userId, status: "ACTIVE" } }),
    prisma.ad.aggregate({ where: { userId }, _sum: { viewCount: true } }),
  ]);

  const totalViews = totalViewsAgg._sum.viewCount ?? 0;
  const totalPages = Math.max(1, Math.ceil(total / pageSize));

  return (
    <div className="mx-auto max-w-screen-xl px-4 py-6">
      {/* 헤더 + 새 광고 버튼 (기존과 동일) */}
      {/* 요약 카드 3개 (기존과 동일) */}

      {/* 상태 필터 탭 + 검색창 (스탭 전용) */}
      <div className="my-4 flex flex-wrap items-center gap-2">
        <form method="GET" className="flex gap-2">
          <select name="status" defaultValue={statusFilter} className="rounded border bg-background px-3 py-2 text-sm">
            <option value="ALL">전체</option>
            <option value="ACTIVE">게재중</option>
            <option value="EXPIRED">만료</option>
            <option value="CANCELLED">취소</option>
          </select>
          <input
            type="text"
            name="search"
            defaultValue={searchQuery}
            placeholder="제목/사업체명 검색"
            className="rounded border bg-background px-3 py-2 text-sm"
          />
          <Button type="submit" size="sm">검색</Button>
        </form>
      </div>

      {/* 광고 리스트 (스탭 버전: 카드 클릭 → 광고 상세로) */}
      <div className="space-y-3">
        {ads.map((ad) => (
          <Card key={ad.id}>
            <CardContent className="py-4">
              {/* 카드 전체 링크를 /business/ads/[id] (광고 상세)로 */}
              <Link href={`/business/ads/${ad.id}`}>
                {/* ... 광고 정보 표시 (기존과 유사하되) ... */}
              </Link>

              {/* 액션 버튼: 업그레이드/연장 없음, 수정/통계/삭제만 */}
              {ad.status === "ACTIVE" && (
                <div className="mt-2 flex flex-wrap items-center gap-2 border-t pt-2">
                  {ad.manualJumpPerDay > 0 && (
                    <JumpButton adId={ad.id} ... />
                  )}
                  <Link href={`/business/ads/${ad.id}/stats`}>
                    <Button size="sm" variant="outline" className="text-xs">통계</Button>
                  </Link>
                  <Link href={`/business/ads/${ad.id}/edit`}>
                    <Button size="sm" variant="ghost" className="text-xs">수정</Button>
                  </Link>
                  {/* 업그레이드/연장 버튼 없음 */}
                  <DeleteAdButton adId={ad.id} isPaidActive={false} />
                </div>
              )}
              {/* EXPIRED 블록도 연장 버튼 없이 통계/삭제만 */}
            </CardContent>
          </Card>
        ))}
      </div>

      {/* 인라인 페이지네이션 (Pagination 컴포넌트 없음) */}
      {total > pageSize && (
        <div className="mt-6 flex items-center justify-center gap-2">
          {currentPage > 1 && (
            <Link href={`/business/dashboard?status=${statusFilter}&search=${searchQuery}&page=${currentPage - 1}`}>
              <Button size="sm" variant="outline">이전</Button>
            </Link>
          )}
          <span className="text-sm text-muted-foreground">
            {currentPage} / {totalPages}
          </span>
          {currentPage < totalPages && (
            <Link href={`/business/dashboard?status=${statusFilter}&search=${searchQuery}&page=${currentPage + 1}`}>
              <Button size="sm" variant="outline">다음</Button>
            </Link>
          )}
        </div>
      )}
    </div>
  );
}
```

**중요**: 스탭 대시보드는 업그레이드/연장 버튼이 **아예 없음**. 카드 전체 클릭 링크도 `/business/ads/${id}/payment` 대신 `/business/ads/${id}` (광고 상세)로 변경.

import bcrypt from "bcryptjs";

async function main() {
  const email = "staff@yeosijob.com";
  const password = process.env.STAFF_INITIAL_PASSWORD;
  if (!password) throw new Error("STAFF_INITIAL_PASSWORD env required");

  if (existing) {

      where: { email },
      data: { isStaff: true, isVerifiedBiz: true },

      password: hash,
      name: "여시잡 스탭",
      businessName: "여시잡",
      role: "BUSINESS",
      isStaff: true,
      isVerifiedBiz: true,
    },
  });
  console.log("Staff account created:", user.email);
}

main().catch(console.error).finally(() => prisma.$disconnect());
